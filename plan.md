# План реализации проекта AdaptiveChatRAG

## 1. Идея и описание
**AdaptiveChatRAG** – самообучающийся RAG-чат-бот с памятью.
Интерактивный бот, который использует RAG и механизмы памяти для постоянного улучшения своих ответов. Поддерживает пользовательскую обратную связь: если бот ошибается, пользователь может это отметить, и система динамически откорректирует свой векторный индекс и параметры поиска.

## 2. Уникальные особенности
- **Память на уровне сеанса:** хранение истории диалога с использованием эмбеддингов для более точного понимания контекста будущих вопросов.
- **Ретроспективная переиндексация:** механизм добавления/удаления документов или изменения их весов в индексе на основе прямых исправлений от пользователя.
- **Оценка в реальном времени:** отслеживание влияния обратной связи на метрики качества (через Open RAG Eval / Ragas).
- **Мультиплатформенность:** конфигурируемый текстовый интерфейс (например, Telegram-бот) и веб-панель для статистики.

## 3. Выбор технологий
- **Фреймворк RAG:** LangChain (работа с агентами, памятью ConversationBuffer).
- **База данных:** Векторная БД (Chroma/Milvus) + реляционная БД (PostgreSQL/SQLite) для сохранения логов, диалогов и обратной связи.
- **Оценка:** Ragas (для фонового отслеживания качества).
- **Интерфейс:** aiogram (Telegram) и FastAPI (API / веб-админка).

## 4. Поэтапный план разработки

### Этап 1. Базовый бот с RAG и памятью
- Интеграция базового пайплайна "Документ -> Индекс -> Поиск -> Генерация".
- Реализация сессионной памяти для поддержания контекста диалога.

### Этап 2. Механизм обратной связи (Feedback Loop)
- Добавление интерфейса обратной связи в бот (кнопки лайк/дизлайк, текстовые уточнения).
- Логирование фидбека в реляционную БД.

### Этап 3. Адаптивный поиск и корректировка
- Разработка механизма обновления базы: если документ вызвал "дизлайк", его вес в поиске временно снижается или он отправляется на ручную проверку.
- Использование подходов Self-RAG/Corrective RAG.

### Этап 4. Тестирование и фоновая оценка
- Проверка работоспособности: как бот "запоминает" ошибки в течение сессии.
- Периодический прогон метрик Ragas по логам чатов.

### Этап 5. Документация и публикация
- Подробное описание архитектуры обратной связи.
- Инструкция по запуску и настройке ботов.
- Опубликовать код на GitHub.
